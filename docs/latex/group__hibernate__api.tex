{\footnotesize\ttfamily uint32\+\_\+t Hibernate\+Bat\+Check\+Done (\begin{DoxyParamCaption}\item[{void}]{ }\end{DoxyParamCaption})}

Determines whether or not a forced battery check has completed.

This function determines whether the forced battery check initiated by a call to the \mbox{\hyperlink{group__hibernate__api_ga74ba4cc82387398559b5e6797ef2424f}{Hibernate\+Bat\+Check\+Start()}} function has completed. This function returns a non-\/zero value until the battery level check has completed. Once this function returns a value of zero, the Hibernation module has completed the battery check and the \mbox{\hyperlink{group__hibernate__api_gaf172d65cf6fa81e70561f004ec8794cf}{Hibernate\+Int\+Status()}} function can be used to check if the battery was low by checking if the value returned has the {\bfseries{HIBERNATE\+\_\+\+INT\+\_\+\+LOW\+\_\+\+BAT}} set.

\begin{DoxyReturn}{Returns}
The value is zero when the battery level check has completed or non-\/zero if the check is still in process. 
\end{DoxyReturn}

{\footnotesize\ttfamily void Hibernate\+Bat\+Check\+Start (\begin{DoxyParamCaption}\item[{void}]{ }\end{DoxyParamCaption})}

Forces the Hibernation module to initiate a check of the battery voltage.

This function forces the Hibernation module to initiate a check of the battery voltage immediately rather than waiting for the next check interval to pass. After calling this function, the application should call the \mbox{\hyperlink{group__hibernate__api_gaa3ffe177224a1f3bcc3deb9d134cc14f}{Hibernate\+Bat\+Check\+Done()}} function and wait for the function to return a zero value before calling the \mbox{\hyperlink{group__hibernate__api_gaf172d65cf6fa81e70561f004ec8794cf}{Hibernate\+Int\+Status()}} to check if the return code has the {\bfseries{HIBERNATE\+\_\+\+INT\+\_\+\+LOW\+\_\+\+BAT}} set. If {\bfseries{HIBERNATE\+\_\+\+INT\+\_\+\+LOW\+\_\+\+BAT}} is set, the battery level is low. The application can also enable the {\bfseries{HIBERNATE\+\_\+\+INT\+\_\+\+LOW\+\_\+\+BAT}} interrupt and wait for an interrupt to indicate that the battery level is low.

\begin{DoxyNote}{Note}
A hibernation request is held off if a battery check is in progress.
\end{DoxyNote}

None.

This function returns the current date and time in the structure provided by the {\itshape ps\+Time} parameter. Regardless of the calendar mode, the {\itshape ps\+Time} parameter uses a 24-\/hour representation of the time. This function can only be called when the Hibernation module is configured in calendar mode using the \mbox{\hyperlink{group__hibernate__api_ga97a504a0525e66367e32b45cecd03f0d}{Hibernate\+Counter\+Mode()}} function with one of the calendar modes.

The only case where this function fails and returns a non-\/zero value is when the function detects that the counter is passing from the last second of the day to the first second of the next day. This exception must be handled in the application by waiting at least one second before calling again to get the updated calendar information.

\begin{DoxyNote}{Note}
The hibernate calendar mode is not available on all Tiva devices. Please consult the data sheet to determine if the device you are using supports this feature in the Hibernation module.
\end{DoxyNote}
\begin{DoxyReturn}{Returns}
Returns zero if the time and date were read successfully and returns a non-\/zero value if the {\itshape ps\+Time} structure was not updated.

{\footnotesize\ttfamily void Hibernate\+Request (\begin{DoxyParamCaption}\item[{void}]{ }\end{DoxyParamCaption})}

Requests hibernation mode.

This function requests the Hibernation module to disable the external regulator, thus removing power from the processor and all peripherals. The Hibernation module remains powered from the battery or auxiliary power supply.

The Hibernation module re-\/enables the external regulator when one of the configured wake conditions occurs (such as RTC match or external {\bfseries{WAKE}} pin). When the power is restored, the processor goes through a power-\/on reset although the Hibernation module is not reset. The processor can retrieve saved state information with the \mbox{\hyperlink{group__hibernate__api_ga89c00c7f5d7e68f5e68308017b702203}{Hibernate\+Data\+Get()}} function. Prior to calling the function to request hibernation mode, the conditions for waking must have already been set by using the \mbox{\hyperlink{group__hibernate__api_gab2a9a40792b72e0f7d1f73c62e03de42}{Hibernate\+Wake\+Set()}} function.

Note that this function may return because some time may elapse before the power is actually removed, or it may not be removed at all. For this reason, the processor continues to execute instructions for some time, and the caller should be prepared for this function to return. There are various reasons why the power may not be removed. For example, if the \mbox{\hyperlink{group__hibernate__api_gabd9581d7a8446c62333daa5a83e0920b}{Hibernate\+Low\+Bat\+Set()}} function was used to configure an abort if low battery is detected, then the power is not removed if the battery voltage is too low. There may be other reasons related to the external circuit design, that a request for hibernation may not actually occur.

For all these reasons, the caller must be prepared for this function to return. The simplest way to handle it is to just enter an infinite loop and wait for the power to be removed.

The format of the returned {\itshape pui32\+RTC} data is dependent on the configuration of the RTC within the Hibernation module. signal 1
- \b HIBERNATE_TAMPER_EVENT_2 indicates a tamper event was triggered on I/O
signal 2
- \b HIBERNATE_TAMPER_EVENT_3 indicates a tamper event was triggered on I/O
