{\footnotesize\ttfamily void AESAuth\+Length\+Set (\begin{DoxyParamCaption}\item[{uint32\+\_\+t}]{ui32\+Base,  }\item[{uint32\+\_\+t}]{ui32\+Length }\end{DoxyParamCaption})}

Sets the authentication data length in the AES module.


\begin{DoxyParams}{Parameters}
{\em ui32\+Base} & is the base address of the AES module. \\
\hline
{\em ui32\+Length} & is the length in bytes.\\
\hline
\end{DoxyParams}
This function is only used to write the authentication data length in the combined modes (GCM or CCM) and XTS mode. Supported AAD lengths for CCM are from 0 to (2$^\wedge$16 -\/ 28) bytes. For GCM, any value up to (2$^\wedge$32 -\/ 1) can be used. For XTS mode, this register is used to load j. Loading of j is only required if j != 0. j represents the sequential number of the 128-\/bit blocks inside the data unit. Consequently, j must be multiplied by 16 when passed to this function, thereby placing the block number in bits \mbox{[}31\+:4\mbox{]} of the register.

When this function is called, the engine is triggered to start using this context for GCM and CCM.

\begin{DoxyReturn}{Returns}
None 
\end{DoxyReturn}

This function configures the AES module based on the specified parameters. It does not change any DMA-\/ or interrupt-\/related parameters.

The ui32\+Config parameter is a bit-\/wise OR of a number of configuration flags. The valid flags are grouped based on their function.

The direction of the operation is specified with only of following flags\+:


\begin{DoxyItemize}
\item {\bfseries{AES\+\_\+\+CFG\+\_\+\+DIR\+\_\+\+ENCRYPT}} -\/ Encryption mode
\item {\bfseries{AES\+\_\+\+CFG\+\_\+\+DIR\+\_\+\+DECRYPT}} -\/ Decryption mode

\item {\bfseries{AES\+\_\+\+CFG\+\_\+\+KEY\+\_\+\+SIZE\+\_\+256\+BIT}} -\/ Key size of 256 bits
