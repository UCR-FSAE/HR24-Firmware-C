Configures the USB device mode response to LPM requests.


\begin{DoxyParams}{Parameters}
{\em ui32\+Base} & specifies the USB module base address. \\
\hline
{\em ui32\+Config} & is the combination of configuration options for LPM transactions in device mode.\\
\hline

This function sets the global configuration options for LPM transactions in device mode and must be called before ever calling \mbox{\hyperlink{group__usb__lpm_ga2b9bf534386fa02ad13b0ada5e9d822a}{USBDev\+LPMEnable()}} to set the configuration for LPM transactions. The configuration options in device mode are specified in the {\itshape ui32\+Config} parameter and include one of the following\+:


\begin{DoxyItemize}
\item {\bfseries{USB\+\_\+\+DEV\+\_\+\+LPM\+\_\+\+NONE}} disables the USB controller from responding to LPM transactions.
\item {\bfseries{USB\+\_\+\+DEV\+\_\+\+LPM\+\_\+\+EN}} enables the USB controller to respond to LPM and extended transactions.
\item {\bfseries{USB\+\_\+\+DEV\+\_\+\+LPM\+\_\+\+EXTONLY}} enables the USB controller to respond to extended transactions, but not LPM transactions.
\end{DoxyItemize}

The {\itshape ui32\+Config} option can also optionally include the {\bfseries{USB\+\_\+\+DEV\+\_\+\+LPM\+\_\+\+NAK}} value to cause the USB controller to NAK all transactions other than an LPM transaction once the USB controller is in LPM suspend mode. If this value is not included in the {\itshape ui32\+Config} parameter, the USB controller does not respond in suspend mode.

The USB controller does not enter LPM suspend mode until the application calls the \mbox{\hyperlink{group__usb__lpm_ga2b9bf534386fa02ad13b0ada5e9d822a}{USBDev\+LPMEnable()}} function.
